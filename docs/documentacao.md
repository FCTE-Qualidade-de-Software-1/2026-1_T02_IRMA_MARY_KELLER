# 📚 Documentação

## Organização dos Documentos

Esta seção centraliza o acesso a toda a documentação do projeto, organizada por fases e tópicos.

### 📖 Guias Principais

#### 1. **Planejamento e Especificação**
- Escopo do projeto
- Características de qualidade
- Modelo de avaliação (ISO/IEC 25010)
- Requisitos e restrições

#### 2. **Execução da Avaliação**
- Casos de teste
- Resultados de testes
- Métricas de qualidade
- Análise de conformidade

#### 3. **Relatórios e Resultados**
- Relatório final de qualidade
- Recomendações de melhoria
- Plano de ação

#### 4. **Evidências e Anexos**
- Screenshots de testes
- Registros de execução
- Tabelas de compatibilidade
- Logs de análise

## 🗂️ Estrutura de Diretórios

```
docs/
├── index.md                 # Página inicial
├── sobre.md                 # Informações do projeto
├── integrantes.md           # Equipe
├── documentacao.md          # Este arquivo
├── fases/
│   ├── fase_1/             # Planejamento
│   ├── fase_2/             # Especificação
│   ├── fase_3/             # Preparação
│   └── fase_4/             # Execução
└── anexos/
    ├── templates/          # Modelos de testes
    ├── metricas/           # Planilhas de métricas
    └── evidencias/         # Capturas e registros
```

## 📋 Como Contribuir

1. Crie um arquivo markdown na pasta correspondente
2. Atualize o `mkdocs.yml` com a nova página
3. Siga o padrão de formatação existente
4. Commit e push para o repositório

## 📝 Padrão de Documentação

### Cabeçalhos
```markdown
# Título Principal (H1)
## Título Secundário (H2)
### Título Terciário (H3)
```

### Listas e Tabelas
```markdown
- Item 1
- Item 2
  - Sub-item 2.1

| Coluna 1 | Coluna 2 |
|----------|----------|
| Valor 1  | Valor 2  |
```

### Ênfase
```markdown
**Negrito**
*Itálico*
~~Riscado~~
```
